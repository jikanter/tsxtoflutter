# `flutter_app/android/` — hand-maintained Android Gradle assets

The full `android/` tree is regenerated by `flutter create --platforms=android .`,
but the files listed below are committed and **must not be overwritten**:

| File                                    | Purpose                                                    |
|-----------------------------------------|------------------------------------------------------------|
| `app/build.gradle.kts`                  | compileSdk=36, targetSdk=36, minSdk=24, R8, AAB, signing.  |
| `app/src/main/AndroidManifest.xml`      | `enableOnBackInvokedCallback="true"` for predictive back.  |
| `app/proguard-rules.pro`                | R8 keep rules per closed-widget-catalog plugin.            |
| `key.properties.template`               | Template (no secrets); copy to `key.properties` locally.   |

`key.properties` itself is gitignored at the repo root and must never be
committed.

## After running `flutter create`

`flutter create` will scaffold the rest of the `android/` tree (Kotlin Runner
class, Gradle wrapper, `gradle.properties`, etc.). When it asks whether to
overwrite an existing file, choose **no** for the four files above. If
`build.gradle.kts` was overwritten, restore it from git: `git checkout
flutter_app/android/app/build.gradle.kts`.

## Signing

CI: set `ANDROID_KEYSTORE_PATH`, `ANDROID_KEYSTORE_PASSWORD`, `ANDROID_KEY_ALIAS`,
`ANDROID_KEY_PASSWORD` env vars; the build.gradle.kts falls back to env when
`key.properties` is missing.

Local dev: copy `key.properties.template` to `key.properties` (already
gitignored) and fill in your upload-keystore details, OR rely on the default
debug signing.

## 16 KB page-size alignment

Android 15+ requires 64-bit native libraries to be 16 KB page-aligned. AGP 8.5+
aligns by default. The build.gradle.kts disables legacy JNI packaging
(`useLegacyPackaging = false`); CI should additionally run
`zipalign -c -P 16 -v 4 <aab>` and fail loudly if any plugin is misaligned.
